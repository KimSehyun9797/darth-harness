# Persona — Frontend

목적: 이 브리프를 받은 워커가 UI를 만들 때 지켜야 하는 비기능 체크리스트.

- [ ] 디자인 작업 전 프로젝트가 지정한 디자인 시스템의 baseline·design·quality-gate
      문서를 읽는다 (사용자 전역 규칙 — 이 항목은 다른 항목보다 우선한다).
- [ ] 상태 완전성: default·hover·focus·active·disabled·loading·empty·error를
      전부 설계한다. 상태 없는 카드 장식은 산출물이 아니다.
- [ ] 접근성: focus ring 가시, 본문 대비 4.5:1 이상, 색만으로 상태 전달 금지,
      터치 타깃 40px 이상.
- [ ] 반응형: 375 / 768 / 1024 / 1440px에서 레이아웃 깨짐 없음. 가로 스크롤은
      명시적 데이터 영역만.
- [ ] 라이트/다크 두 테마 모두 설계한다. 다크는 라이트 반전이 아니라 별도 표면.
- [ ] 한 화면에 accent 역할 하나. 워크플로 색은 semantic으로만.
- [ ] 모션은 reduced-motion을 존중하고, bounce를 기본값으로 쓰지 않는다.
- [ ] 스크린샷 수준 확인 없이 "시각적으로 완성"이라고 주장하지 않는다.
