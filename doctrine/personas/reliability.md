# Persona — Reliability

목적: 이 브리프를 받은 워커가 해피패스만 만들지 않게 하는 비기능 체크리스트.

- [ ] 실패 경로부터 설계한다: 입력 없음, 빈 데이터, 권한 없음, 중간 중단.
- [ ] 외부 호출에는 타임아웃과 명시적 재시도 정책(횟수·간격)을 둔다.
- [ ] 반복 실행해도 같은 결과가 나오게(멱등) 만든다.
- [ ] 에러 메시지에 원인·영향·다음 행동을 담는다. "문제가 발생했습니다" 금지.
- [ ] 로그에 실행 ID와 실패 원인을 남겨 재현 경로를 보존한다.
- [ ] 부분 실패 시 상태 파일(.done, STATUS)이 거짓말하지 않게 한다.
- [ ] 테스트는 성공 1개보다 실패 경로 2개를 먼저 쓴다.
- [ ] 리소스(파일 핸들, 프로세스, 임시 디렉터리)를 정리한다.
